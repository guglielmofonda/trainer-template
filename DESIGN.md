# Design

The visual system for the trainer — codename **"Ledger."** The north star: **a serious
lab report for your own strength.** Paper and ink, hairline rules, sharp corners,
numbers-first. Restraint over decoration. Read alongside [PRODUCT.md](PRODUCT.md) —
this file answers *how it looks*; that one answers *who/what/why*.

## Theme

Light, by deliberate choice — not "light to be safe." The scene: a serious
self-coaching lifter auditing a periodized block at a desk, or checking the next set
on their phone under bright gym lighting. They trust numbers, not vibes. The right
reference is a **scientific journal crossed with measurement gear** — a training
logbook or a spec sheet — not a glowing app. The interface should read like something
printed and precise, that a coach would trust at a glance.

- **Mode:** light only (for now). Cool-neutral paper — never a warm cream/parchment.
  The whole system is token-driven, so a true-black dark variant is a token override,
  not a rebuild.
- **Color strategy:** near-monochrome chrome. Everything structural is paper, ink,
  and gray. A single reserved **signal accent** (vermilion) carries the *live* and
  *danger* moments only. A small, clearly-scoped semantic ramp carries *data state*
  (volume verdicts) — earned because the engine genuinely has five volume verdicts.
- **Atmosphere:** flat. Structure comes from **hairline rules, sharp corners, and
  whitespace** — never shadows, glows, gradients, or backdrop-blur. Depth is
  typographic (scale, weight, tabular alignment), not physical.

## Color

All tokens are OKLCH. Defined in `app/globals.css` under `@theme`.

### Paper & ink — cool neutral (hue ≈ 258, chroma ≈ 0)

| Token | OKLCH | Role |
|---|---|---|
| `--color-bg` | `oklch(0.986 0.0015 255)` | Paper — app + panel background |
| `--color-surface` | `oklch(0.969 0.002 255)` | Sidebar rail, zebra, chart plate |
| `--color-surface-2` | `oklch(0.944 0.003 256)` | Input wells, hover, bar tracks |
| `--color-surface-3` | `oklch(0.908 0.004 258)` | Active / selected fill |
| `--color-border` | `oklch(0.858 0.004 258)` | Standard hairline rule |
| `--color-border-soft` | `oklch(0.916 0.003 258)` | Faint divider |
| `--color-border-strong` | `oklch(0.7 0.006 260)` | Emphasis rule / hover border |
| `--color-ink-strong` | `oklch(0.16 0.008 265)` | Headings, key numbers (max contrast) |
| `--color-ink` | `oklch(0.25 0.006 264)` | Primary text |
| `--color-muted` | `oklch(0.448 0.006 262)` | Secondary text (≥ 4.5:1 on paper) |
| `--color-faint` | `oklch(0.58 0.006 262)` | Captions / labels — large or non-essential text only |

### Signal accent — reserved vermilion

| Token | OKLCH | Role |
|---|---|---|
| `--color-accent` | `oklch(0.55 0.204 27)` | The one signal. Live readouts, focus ring, links, danger. |
| `--color-accent-strong` | `oklch(0.485 0.213 27)` | Accent hover |
| `--color-accent-ink` | `oklch(0.99 0.002 250)` | Text on an accent fill |

Red is **reserved and meaningful** — it appears on the live e1RM recompute (the engine
visibly responding), the current strength trend line, danger verdicts (under MEV /
over MRV), focus rings, links, and error states. **Never chrome decoration.** Primary
actions and active nav are **ink (black)**, not red — so when red appears, it means
something. On a page where nothing is wrong, there is almost no color.

### Semantic data ramp — state only, not chrome

Used inside the planning check, volume bars, badges, and progression results.

| Token | OKLCH | Meaning |
|---|---|---|
| `--color-success` | `oklch(0.47 0.115 152)` | Productive / on-track. Deep, serious green. |
| `--color-warning` | `oklch(0.55 0.14 56)` | Running hot — approaching MRV. Burnt ochre. |
| `--color-danger` | `oklch(0.52 0.2 27)` | Below MEV or above MRV. Red (shares the accent hue). |
| `--color-info` | `oklch(0.5 0.088 250)` | Deload / maintenance. Slate. |

Verdict → color: `productive → success`, `high → warning`, `over/under → danger`,
`maintenance → info`, deload → info. **Color is never the only signal** — verdicts
always pair with a text label (a ruled caps tag) and position.

### Composition (share-of-sets chart)

**Not categorical.** "Share of weekly sets" is proportion data, so the composition bar
uses a single **ink lightness ramp** (grayscale) — darkest = largest contributor,
fading to mid-gray — generated by `compositionShades(n)` in `components/ui.tsx`.
Segments and legend are ranked largest-first. A full-spectrum palette read as a
rainbow toy; one grayscale ramp reads as one calm instrument. Every segment carries a
muscle label and % in the legend, so color is never the only signal.

## Typography

Three families on contrast axes (serif display / sans UI / mono data):

- **Fraunces** — the editorial display voice: page titles and the brand. `--font-serif`
  (via `next/font/google`). Used sparingly — authority, not decoration. Class: `.display`.
- **Geist Sans** — UI, section titles, labels, prose, buttons. `--font-sans`.
- **Geist Mono** — **every number and readout**: loads, reps, RIR, e1RM, intensity,
  volume, week labels, set inputs. `--font-mono`. Always `font-variant-numeric:
  tabular-nums`. Class: `.tabular`.

Product register → fixed rem scale (not fluid clamp), tight ratio (~1.2). Three tiers
of hierarchy: **serif display** (page title, brand) → **sans section titles**
(PanelHeader, 15px semibold) → **small-caps `.label`** (spec captions, table headers,
the ledger voice — replaces the pill/eyebrow reflex).

Headings: `text-wrap: balance`. Prose (Theory): serif headings, caps `.label`-style
h3, ~68ch measure, `text-wrap: pretty`.

## Spacing & Layout

- **App shell:** fixed sidebar (`232px`, on `--color-surface`) + scrollable `<main>`
  on paper. Sidebar collapses to a top bar under `768px`.
- Spacing scale (4px base): 2, 4, 6, 8, 12, 16, 20, 24, 32, 40, 48. Vary it for
  rhythm; avoid uniform gaps.
- **Radii:** `--radius-sm 1px`, `--radius-md 2px`, `--radius-lg 3px`. Sharp —
  instrument, never bubbly. Pills are banned; use ruled caps tags. The only true
  circle is a status dot / verdict mark.
- **Rules over cards.** Panels are flat, hairline-bordered fields on paper with a
  header/body rhythm; never a shadowed rounded card, never an icon+heading+text card
  grid, never nested cards.
- **Tables are first-class** (Program grid, periodization, planning check, progress).
  Hairline rules, small-caps column headers, right-aligned tabular-mono numerics,
  generous row height for mid-gym tapping.
- **z-index scale** (semantic, never 9999): dropdown 100 · sticky 200 · backdrop 300
  · modal 400 · toast 500 · tooltip 600.

## Components

Live in `components/ui.tsx` (+ `Sidebar.tsx`, the loggers, the chat). Every
interactive component ships **default, hover, focus-visible, active, disabled** and,
where relevant, **loading / error / empty**.

- **Panel** — flat hairline-ruled field on paper with optional `PanelHeader` (title +
  meta/right slot). The base container. (`lit` prop retained for API compat; no glow.)
- **Readout** — the signature: big tabular-mono value + `.label` caption + optional
  unit / sub. Ink by default; `accent` (red) only for live strength readouts. Used for
  e1RM, prescription load, weekly totals.
- **Badge** — a **ruled caps tag** (sharp, hairline box, uppercase, tracked). Tone is
  carried by a small square mark, not text color, so the label stays high-contrast ink
  and color is never the only signal.
- **Button** — `primary` (ink fill, paper text), `secondary` (paper + hairline
  border), `ghost`. Sizes sm/md. Focus-visible accent ring.
- **DayTabs / WeekRail** — the day/week selector: active segment is an **ink (black)
  fill**, not a colored pill.
- **NumberField / OpeningSingleField** — numeric inputs with a unit suffix, accent
  focus ring, mono value.
- **VolumeBar** — a precision gauge: MEV/MAV/MRV landmark bar with an ink-tinted MAV
  band, achieved fill colored by verdict, ink MEV/MRV ticks, accessible label.
  Squared (1px), animated width on mount (instant under reduced-motion).
- **Icons:** `lucide-react`, one set only. No emoji in UI chrome. Icon-only controls
  carry an `aria-label`. The favicon/brand mark is a dark ledger square with a serif
  monogram (the athlete's initials, from `data/profile.json`) and the single red accent on the underline.

## Motion

Product motion: conveys state, never decorates. Tokens in `globals.css`.

- Durations: `--dur-fast 110ms`, `--dur 170ms`, `--dur-slow 260ms`.
- Easing: `--ease-out: cubic-bezier(0.22, 1, 0.36, 1)` (exponential ease-out, no
  bounce/elastic).
- Where motion lives: nav hover/active, panel expand/collapse, focus rings, the
  **live readout transition** when the session logger recomputes the prescription from
  a typed opening single (a brief red flash — the one place the accent moves), and
  volume-bar fills on mount.
- **No orchestrated page-load sequence.** The app loads into a task.
- **`prefers-reduced-motion: reduce`** → crossfade or instant for every animation;
  bars render at final width immediately.

## Anti-patterns (enforced)

Shared Impeccable bans + product bans apply. Specifically rejected here — and note the
**dark AI-dashboard is now the primary anti-reference:** warm-graphite-on-black, a
molten-amber accent, glowing/lit panels, backdrop-blur glass, drop shadows, gradients,
rounded (≥8px) cards, the hero-metric tile strip, a categorical rainbow chart, pills
as status chips, per-section uppercase eyebrows, emoji as UI icons, display fonts in
data/labels, and any color used as the sole carrier of meaning. If it looks
auto-generated from "dark dashboard," it's wrong.
